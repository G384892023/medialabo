数当てゲーム

このゲームはランダムで決められた1~10までの数字を予測して当てるというものだ。

答えはランダムに設定され、入力した値よりも答えが大きければ「もっと大きいです」、

反対に小さい場合は「もっと小さいです」とされるため、そこから予測することで正答率上げることができる。

挑戦可能回数は3回まで。

3回目までに数を当てれなかった場合「ゲームは終了しています」とともに答えが出力される。

同様に、正解した後や、制限以上の回答を行った場合も同じように出力される。

プログラム自体は簡単だが、（「」n「」）のようなとき、nだけを書き換える方法がわからなかった。

()の中を全て消した後もう一度カッコを作り直す方法しか思いつかなかった。

各授業回の学習事項

第1回 WWW と HTML

HTML による Web ページの構造を学び、さまざまなHTMLの要素を学び、実践を通して、記憶に定着させる作業を行なった。具体的には、ハイパーリンクの作成方法、文字のサイズ調整などの作成方法を学んだ。また、ステージの変更、コミット、プッシュを忘れないように注意。

演習の成果：演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSS作成について学んだ。CSSには、文字の色を変更したり、文字の周りや下のぶぶに実線や点線を引いたり、書いた文章全体を回転させたり、色にグラデーションをつけたりなどが行えるようだ

演習の成果：ウェブブラウザの紹介

第3回 JavaScript の基本

ボタンを押した時にあるプログラムを実行したり、コンソール上に特定の文字を表示したりするシステムをJavaScriptを使い出力できるようになった。JavaScriptは扱うのが今回初めてで、構文や働きの意味がわからなかったのでそれを学ぶことに苦労した

演習の成果：数当てゲーム

演習の成果：学校

第4回 DOM と JavaScript

Web コンテンツの内容を読み取ったり、コンテンツに変更を加えたりする処理を行うDOMについて学び学んだ。これにより作成できるものの幅が広がった。ページの書き換えは特に重要で、同じページ内での瞬間的な画面の変更を起こすことができるようになった。

演習の成果：修正挿入

第5回 イベント処理

イベント処理には、ボタンをクリック・キーボードのキーを押す・Webページのリロードなどがあり、画面上の変化をさまざまな変化を画面上に発生させれるようになった。これにより、持続的な画面の動きなども作れるようになった。

演習の成果：イベントリスト
